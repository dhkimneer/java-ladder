# 사다리 게임
## 2단계 - 사다리(생성)
1. 기능 요구사항
   1. 사다리 게임에 참여하는 사람의 이름을 최대 5글자까지 부여
   2. 사다리 출력 시 사람 이름도 같이 출력
   3. 사람 이름은 쉼표를 기준으로 구분
   4. 사다리 폭도 넓어져야 한다.
   5. 라인이 겹치지 않아야 한다.

2. 프로그래밍 요구 사항
   1. 스트림, 람다 적용 프로그래밍
   2. 모든 엔티티를 작게 유지한다.

3. 힌트
   1. 한 라인의 좌표 값을 가지는 객체를 추가해 구현 - ArrayList<Line>

## 3단계 - 사다리(게임 실행)
1. 기능 요구사항
   1. 사다리 실행 결과 출력
   2. 개인별 이름 입력 시 개인별 결과 출력
   3. all 입력 시 전체 참여자의 실행 결과 출력

2. 프로그래밍 요구 사항 (추가)
   1. 3개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다. - 각 객체가 2개 이하의 인스턴스 변수만을 가지도록 구현

3. 힌트
   1. 로직 구현을 위해 필요한 데이터를 가지는 객체를 분리하기 위해 노력

## 4단계 - 사다리(리팩토링)
1. 기능 요구사항
   - 3단계와 같음.

2. 적용 사항
   - 인터페이스 기반 구현
